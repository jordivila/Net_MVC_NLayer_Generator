This widget is the first one of a serie intended to be lightweight widgets to use among google maps API reference.

Overview

jQuery UI widget for creating themable google maps direction service utility.
Use this widget for computing directions between two places by specifying location of origin, destination, preferred unit system and a type of routing.
The widget definition includes some english literals. These literals have been defined in the same file so that you can use/change them easily.
        
var uigMapDirectionsTxt = {
    uiTitleBox: "Get Directions"
    , uiFrom: "From"
    , uiTo: "To"
    , uiByCar: "By Car"
    , uiByWalk: "Walking"
    , uiBicycling: "Bicycling"
    , uiUnitImperial: "Imperial"
    , uiUnitMetric: "Metric"
    , uiPrint: "Print"
    , uiClear: "Clear"
    , uiGo: "Go"
    , uiExplain: 'Use this service for computing directions between two places ...'
};

Dependencies:

Google Maps Api v3
ui.core.js
ui.widget.js

Options

  • map

    Type:
    google.maps.Map

    Required google.maps.Map object reference.

    Code examples

    Initialize the widget with the map option specified. 
    jQuery('.selector').gMapDirections({ map: urMap });
    Get or set the map option, after init. 
    
//getter
var oGoogleMap = jQuery('.selector').gMapDirections('option', 'map');
//setter
jQuery('.selector').gMapDirections('option', 'map', googleMapObject);

  • uiTemplate

    Type:
    string

    In case default template does not meet your spectations you can change it by passing new template in options object.
    Just keep in mind there are some html objects needed for the widget:

    • div.ui-gMapDirections-buttons: buttons container
    • button.ui-widget-dirGo: button needed for triggering search service
    • button.ui-widget-clear: button needed to clear previous search results
    • button.ui-widget-print:button needed to print previous results
    • div.ui-gMapDirections-closeHandler
    • div.ui-gMapDirections-info: search service result container
    • div.ui-gMapDirections-dragHandler
    • input.ui-gMapDirections-from
    • input.ui-gMapDirections-to
    • select.ui-gMapDirections-travelMode
    • select.ui-gMapDirections-unit

    Code examples

    Initialize the widget with height option specified. 
    jQuery('.selector').gMapDirections({ uiTemplate: function(){ return "yourHtmlCode";} });

Events

  • close

    Type:
    close

    This event is triggered when the user clicks the close handler

    Code examples

    Supply a callback function to handle the onSelect event as an init option. 
    jQuery('.selector').gMapDirections({ close: function(e, ui) { ... }});
    Bind to the close event by type: gMapDirectionsclose. 
    jQuery('.selector').bind('gMapDirectionsclose', function(event, ui) {...});

Methods

  • destroy

    Signature:
    .gMapDirections('destroy')

    Remove the gMapDirections functionality completely. This will return the element back to its pre-init state.
